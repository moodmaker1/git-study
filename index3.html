🚀 깃허브 작업 과정 완벽 가이드 🚀

처음부터 끝까지 단계별로 따라하기!
1
🌐 깃허브에서 리포지토리 만들기
📝 뭐하는 단계인가요?
온라인에 내 프로젝트를 담을 빈 폴더를 만드는 거예요!
🔧 하는 방법:
1. GitHub.com에 로그인
2. 오른쪽 위 "+" 버튼 클릭
3. "New repository" 선택
4. 프로젝트 이름 입력
5. "Create repository" 클릭
💡 꿀팁: 프로젝트 이름은 영어로, 띄어쓰기 대신 하이픈(-)을 사용하세요!
⬇️
2
📁 로컬(내 컴퓨터)에 폴더 만들기
📝 뭐하는 단계인가요?
내 컴퓨터에 실제로 작업할 폴더를 만드는 거예요!
🔧 하는 방법:
1. 바탕화면이나 원하는 위치에서 우클릭
2. "새 폴더" 선택
3. 깃허브 프로젝트와 같은 이름으로 폴더명 입력
💡 꿀팁: 폴더 이름도 깃허브와 똑같이 만들면 헷갈리지 않아요!
⬇️
3
💻 VS Code로 폴더 열고 터미널 사용
📝 뭐하는 단계인가요?
만든 폴더를 코딩 프로그램으로 열고, 명령어를 입력할 창을 여는 거예요!
🔧 하는 방법:
1. VS Code 실행
2. "File" → "Open Folder" → 만든 폴더 선택
3. 상단 메뉴에서 "Terminal" → "New Terminal" 클릭
🎯 이제 깃허브와 내 컴퓨터를 연결할 준비가 됐어요!
⬇️
4
🔗 깃허브 명령어 복사해서 터미널에 입력
📝 뭐하는 단계인가요?
깃허브에서 제공하는 연결 명령어를 복사해서 붙여넣는 거예요!
이렇게 하면 README.md 파일이 내 폴더에 생겨요!
🔧 하는 방법:
1. 깃허브 리포지토리 페이지로 가기
2. 초록색 "Code" 버튼 클릭
3. "...or create a new repository on the command line" 부분 복사
4. VS Code 터미널에 붙여넣기 후 Enter
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/사용자명/프로젝트명.git
git push -u origin main
💡 README.md가 뭔가요?
프로젝트를 설명하는 파일이에요! 마크다운 형식으로 쓰여져 있어요.
⬇️
5
🌿 브랜치 만들기 (보통 develop)
📝 뭐하는 단계인가요?
안전하게 작업할 수 있는 새로운 작업 공간을 만드는 거예요!
develop은 개발용 브랜치라는 뜻이에요.
🔧 하는 방법:
터미널에 다음 명령어 입력:
git checkout -b develop
💡 왜 develop 브랜치를 만드나요?
• main: 완성된 코드만 보관
• develop: 개발하면서 실험하는 공간
이렇게 나누면 안전해요!
⬇️
6
⚒️ 작업하고 커밋하기
📝 뭐하는 단계인가요?
실제 코딩을 하고, 작업한 내용을 저장하는 거예요!
🔧 하는 방법:
1. 코드 작성 (파일 수정/추가)
2. 터미널에서 커밋하기:
git add .
git commit -m "작업 내용을 간단히 설명"
💡 커밋 메시지 예시:
• "로그인 기능 추가"
• "버그 수정: 회원가입 오류"
• "메인 페이지 디자인 변경"
→ 무엇을 했는지 한눈에 알 수 있게 써주세요!
⬇️
7
🔄 동기화하기 (싱크/푸시)
📝 뭐하는 단계인가요?
내 컴퓨터의 작업 내용을 깃허브에 업로드하는 거예요!
🔧 하는 방법:
터미널에서:
git push origin develop
💡 push가 뭔가요?
내 컴퓨터 → 깃허브로 파일을 업로드하는 것!
이제 인터넷에서 내 작업을 볼 수 있어요.
⬇️
8
🔀 깃허브에서 풀 리퀘스트 & 머지하기
📝 뭐하는 단계인가요?
develop 브랜치의 작업을 main 브랜치로 합치는 거예요!
"개발 완료했으니 메인에 추가해주세요!"라고 요청하는 것!
🔧 하는 방법:
1. GitHub.com의 내 리포지토리로 이동
2. "Compare & pull request" 버튼 클릭
3. 제목과 설명 작성
4. "Create pull request" 클릭
5. "Merge pull request" → "Confirm merge" 클릭
🎉 축하해요! 작업이 성공적으로 메인 브랜치에 합쳐졌어요!

🎯 전체 과정 요약

깃허브 리포지토리 생성 → 로컬 폴더 생성 → VS Code로 열기 → 깃 연결 → develop 브랜치 생성 → 작업 & 커밋 → 푸시(동기화) → 풀 리퀘스트 & 머지


이 과정을 반복하면서 프로젝트를 안전하고 체계적으로 개발할 수 있어요! 🚀
