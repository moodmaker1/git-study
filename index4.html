📚 깃허브로 프로젝트 관리하는 법

깃허브는 코드를 저장하고 팀원들과 함께 작업할 수 있는 온라인 공간이에요. main은 완성된 안전한 코드만, develop은 개발 중인 코드들을 모아두는 곳으로 사용해요.

main ──●────────●─────●── (완성된 코드)
╲ ╱ ╱
develop ●──●──●─────●── (개발 중)
╲ ╱ ╲ ╱
feature ● ●── (새 기능)
1
🏗️ 깃허브에 새 프로젝트 만들기
🎯 이 단계가 뭔가요? 온라인에 내 프로젝트를 담을 폴더를 만드는 거예요!
🔧 따라하기:
1. GitHub.com에 로그인
2. 오른쪽 위 "+" 버튼 → "New repository"
3. 프로젝트 이름 입력: my-first-project
4. ✅ "Add a README file" 체크하기 (꼭 해야 해요!)
5. ✅ ".gitignore" 선택 (프로그래밍 언어에 맞게)
6. "Create repository" 버튼 클릭
🌟 실제 예시:
• 프로젝트명: homework-manager
• 설명: "숙제 관리 웹사이트"
• .gitignore: "Node" 선택 (웹사이트를 만든다면)
⬇️
2
📥 내 컴퓨터로 프로젝트 가져오기
🎯 이 단계가 뭔가요? 온라인에 만든 프로젝트를 내 컴퓨터로 복사해오는 거예요!
🔧 따라하기:
1. 작업하고 싶은 곳으로 이동 (예: 바탕화면)
2. 그 폴더에서 "Git Bash" 또는 "터미널" 열기
3. 아래 명령어 입력하기
# 깃허브에서 초록색 Code 버튼 누르고 주소 복사
git clone https://github.com/내아이디/my-first-project.git
cd my-first-project
# 현재 상태 확인해보기
git status
꿀팁: clone하면 프로젝트명으로 폴더가 자동으로 생기고, README.md 파일도 들어있어요!
⬇️
3
💻 VS Code로 프로젝트 열기
🎯 이 단계가 뭔가요? 코딩할 수 있는 프로그램으로 프로젝트를 열어보는 거예요!
🔧 따라하기:
1. VS Code 켜기
2. "파일" → "폴더 열기" → 복사된 프로젝트 폴더 선택
3. "터미널" → "새 터미널" (또는 Ctrl + `)
4. 깃 상태 확인해보기
# 깃 상태 확인
git status
# 어떤 브랜치에 있는지 확인
git branch
🌟 VS Code에서 확인할 것:
• 왼쪽에 README.md 파일이 보여요
• 아래쪽 상태창에 "main" 이라고 나와요
• 왼쪽 깃 아이콘에서 변경사항을 볼 수 있어요
⬇️
4
🌿 개발용 브랜치 만들기
🎯 이 단계가 뭔가요? main을 안전하게 보호하면서 작업할 수 있는 새로운 공간을 만드는 거예요!
# develop 브랜치 만들고 이동하기
git checkout -b develop
# 온라인에도 develop 브랜치 만들기
git push -u origin develop
🎯 브랜치를 왜 나누나요?
• main: 완성되고 확인된 코드만 보관
• develop: 개발하면서 실험하는 공간
• feature: 새로운 기능 하나씩 만들 때
확인방법: VS Code 아래쪽에 "develop"이라고 바뀐 걸 확인하세요!
⬇️
5
⚒️ 새 기능 브랜치에서 작업하기
🎯 이 단계가 뭔가요? 새로운 기능을 만들 때마다 별도의 작업공간을 만들어서 안전하게 개발하는 거예요!
🔧 새 기능 브랜치 만들기:
# develop에서 최신 상태로 업데이트
git checkout develop
git pull origin develop
# 새 기능 브랜치 만들기
git checkout -b feature/login-page
🔧 작업하고 저장하기:
# 파일 수정한 후 저장하기
git add .
git commit -m "로그인 페이지 추가"
# 더 작업한 후
git commit -m "로그인 버튼 스타일 수정"
🌟 브랜치 이름 예시:
• feature/login-page - 로그인 페이지
• feature/shopping-cart - 장바구니 기능
• fix/login-bug - 로그인 버그 수정
커밋 메시지 쓰는 법:
• "로그인 기능 추가" - 새 기능
• "버그 수정: 회원가입 오류" - 버그 고침
• "디자인 변경: 메인페이지 색상" - 스타일 수정
⬇️
6
🔄 온라인에 작업 내용 올리기
🎯 이 단계가 뭔가요? 내 컴퓨터에서 작업한 내용을 깃허브에 업로드해서 다른 사람들도 볼 수 있게 하는 거예요!
# 처음 올릴 때
git push -u origin feature/login-page
# 나중에 추가로 올릴 때는
git push
🎯 주의사항:
• -u는 처음 한 번만 써요
• 그 다음부터는 git push만 하면 됨
• 이제 깃허브에서 내 작업을 볼 수 있어요!
⬇️
7
🔀 병합 요청하기 (Pull Request)
🎯 이 단계가 뭔가요? "제가 만든 기능을 확인해보시고 메인 프로젝트에 합쳐주세요!"라고 요청하는 거예요!

1️⃣ PR 만들기

깃허브에서 "Compare & pull request"

2️⃣ 리뷰 요청

선생님이나 팀장에게 검토 요청

3️⃣ 피드백 받기

수정할 부분이 있으면 고치기

4️⃣ 승인 & 합치기

OK 받으면 develop에 합치기

🔧 PR 내용 작성 예시:
제목: 로그인 페이지 추가

설명:
## 🎯 뭘 만들었나요?
- 사용자가 로그인할 수 있는 페이지
- 이메일과 비밀번호 입력 폼
- 로그인 버튼 디자인

## ✅ 테스트
- 정상적으로 로그인 되는지 확인
- 잘못된 정보 입력시 오류 메시지 확인
PR 설정할 때:
• Base: develop ← Compare: feature/login-page
• 리뷰어: 선생님이나 팀장 지정하기
⬇️
8
✅ 검토 받고 합치기 완료
🎯 이 단계가 뭔가요? 선생님이나 팀장이 내 코드를 확인하고 OK해주면 develop에 합치는 거예요!
🔧 검토 과정:
1. 리뷰어가 코드 확인
2. 수정이 필요하면 의견 남김
3. 수정사항 반영 후 다시 확인
4. 승인받으면 "Merge pull request" 클릭
# 합치기 완료 후 정리하기
git checkout develop
git pull origin develop
# 완료된 기능 브랜치 삭제
git branch -d feature/login-page
🎉 축하합니다!
• 내가 만든 기능이 프로젝트에 추가됐어요
• develop 브랜치에서 최신 코드 받아오기
• 이제 다음 기능을 만들 준비 완료!

╱ ╱
feature/B ●●
